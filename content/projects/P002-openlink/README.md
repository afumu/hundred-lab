# OpenLink

OpenLink 是百创计划记录的第二个正式开源项目。

源码仓库：

- <https://github.com/afumu/openlink>

## 项目定位

OpenLink 是一个让网页版 AI 直接访问本地文件系统和执行命令的学习研究项目。

它的核心思路不是走传统 API 接入，而是通过浏览器扩展拦截网页 AI 输出的工具指令，再把这些指令转发给本地 Go 服务执行，最后把结果返回给 AI。

主链路可以概括为：

```text
AI 网页
  -> 输出 <tool> 指令
  -> Chrome 扩展拦截
  -> 本地 Go 服务执行
  -> 结果返回 AI
```

从产品体验上看，OpenLink 想解决的问题是：

- 如何让网页版 AI 获得本地工具能力
- 如何让 AI 在浏览器环境中读写文件、执行命令、搜索内容
- 如何用更轻的方式，把本地 agent 能力带到 Gemini、AI Studio 这类网页 AI 产品里

## 当前形态

根据当前 GitHub README，OpenLink 由两部分组成：

- 本地 Go 服务
- Chrome 浏览器扩展

用户安装本地服务并连接扩展后，就可以在 Gemini、AI Studio 等平台中初始化工具能力。

目前仓库明确标注了几个重要边界：

- 这是一个学习研究项目，非生产用途
- 当前实测效果并不理想，稳定性和准确性仍有较大局限
- 它通过模拟用户操作驱动网页 AI，不是一个通用 API

这说明 OpenLink 的价值更偏实验性、探索性和方法验证，而不是成熟商用产品。

## 当前支持的能力

README 中已列出的工具能力包括：

- 执行 Shell 命令
- 列目录
- 读取和写入文件
- 搜索文件名和文件内容
- 精确编辑文件
- 获取网页内容
- 向用户提问
- 加载 Skill
- 写入待办事项

此外，它还支持：

- 输入框里的 `/` Skill 快捷补全
- 输入框里的 `@` 文件路径补全
- 本地 Skills 目录扫描和加载
