# Hundred Lab Extractor

这是百创计划中的第一个正式元技能。

它的职责不是解释整个仓库怎么组织，而是专门负责一件事：

把用户给出的真实工作上下文，优先提炼成产品机会，再进一步拆解为项目、技能、文章和视频。

## 这个技能负责什么

它主要负责：

1. 判断一段上下文里最强的产品机会
2. 对同一段内容做五类资产拆解
3. 找出还缺失的关键信息
4. 继续追问，把模糊经验逼近成可沉淀资产
5. 判断该先进入哪个候选池

## 这个技能不负责什么

它不负责：

1. 代替仓库级默认规则
2. 解释整个仓库的目录结构
3. 决定 Git、发布、同步等仓库管理细节

这些内容仍然应该放在：

- [仓库规则](../../../AGENTS.md)
- [工作流](../../../docs/workflow.md)

## 适合什么时候用

当用户给出下面这些内容时，这个技能最有价值：

- 工作上下文
- 问题与修复过程
- 真实工作流
- 产品想法
- 技术复盘
- 方法论碎片
- 代码片段和实现过程
- 内容选题和表达材料

## 当前定位

这个技能是百创计划的提炼引擎，不是仓库说明书。

它会优先把素材看成潜在产品，再把其中可复用的部分继续拆成开源项目、技能、文章和视频。

## 什么时候更新这份技能

这份技能不需要为了每一次小改动都立刻更新。

更适合更新它的情况是：

1. 你已经把某套提炼方法跑顺了
2. 某些判断逻辑已经相对稳定
3. 你希望把它沉淀成正式资产，未来可复用、可开源

如果只是最近几天想调整提问顺序、候选记录方式、升级节奏，优先改：

- [工作流](../../../docs/workflow.md)

如果是想改仓库级默认行为，优先改：

- [仓库规则](../../../AGENTS.md)
