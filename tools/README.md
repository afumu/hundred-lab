# 工具最佳实践

这里用于沉淀常见工具的使用经验、配置方法、工作流和最佳实践。

这个目录不是正式资产区，而是日常工具经验的长期收纳区。某个工具实践如果已经足够稳定、可复用、可公开传播，后续可以再升级为：

- `content/skills/` 中的正式技能
- `content/articles/` 中的文章
- `content/projects/` 中的工具项目
- `content/products/` 中的产品方向
- `content/videos/` 中的演示或教程视频

## 建议记录内容

每个工具可以单独建立一个子目录，建议包含：

- 工具解决什么问题
- 推荐使用场景
- 安装与配置
- 常用命令或操作
- 个人最佳实践
- 常见坑和解决方案
- 可复用模板或配置片段
- 后续可沉淀的资产方向

## 建议目录结构

```text
tools/
├── README.md
├── software/
│   └── tool-name/
│       └── README.md
└── hardware/
    └── tool-name/
        └── README.md
```

## 当前分类

- [软件工具](./software/)：Claude Code、Codex、YepAnywhere 等软件层面的工具与工作流。

## 升级判断

当某个工具实践满足以下条件时，可以提醒是否升级为正式条目：

- 使用频率高
- 流程稳定
- 对效率提升明显
- 能帮助其他 AI 从业者、开发者或内容创作者
- 已经能形成可复用方法、模板、脚本或教程
