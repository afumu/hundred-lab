# Codex 工作流

## 当前定位

Codex 是后续软件层面工作的默认执行工具。

以后仓库中的日常整理、文档沉淀、代码修改、工具实践记录、资产拆解和自动化任务，默认都优先交给 Codex 完成。

Claude Code 可以作为同类 AI 编程工具参考，但当前默认工作流以 Codex 为主。

## 默认策略

- 软件层面的工作优先交给 Codex。
- Codex 不只用于写代码，也用于整理想法、维护文档、沉淀工具最佳实践和推动百创计划资产化。
- 当任务涉及仓库内容变更时，Codex 应遵守仓库规则：先判断是否进入 `ideas/`，再按产品、项目、技能、文章、视频做资产拆解。
- 如果某个 Codex 使用流程被反复验证有效，后续可以升级成正式技能。

## YepAnywhere

YepAnywhere 是用于控制 Codex 的软件。

后续可以继续补充：

- YepAnywhere 如何启动和连接 Codex
- 常用控制方式
- 适合交给 Codex 的任务类型
- 不适合交给 Codex 的任务边界
- YepAnywhere + Codex 的标准工作流
